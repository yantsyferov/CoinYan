# Functional Specification: Per-Category Budget Limits

- **Roadmap Item:** Per-Category Budget Limits — Users set a monthly spending ceiling for any category. The app alerts them when they approach or exceed the limit.
- **Status:** Completed
- **Author:** yantsyferov

---

## 1. Overview and Rationale (The "Why")

Пользователь уже видит, сколько потратил по каждой категории за месяц, но у него нет инструмента, чтобы заранее решить, сколько он готов потратить. Это приводит к тому, что превышение плана замечается постфактум — уже когда деньги ушли.

Функция месячного лимита даёт пользователю возможность установить для каждой категории расходов личную «планку» на месяц. Как только траты приближаются к ней — это становится заметным визуально; как только пользователь пытается добавить расход, который её превышает — он получает явное предупреждение. При этом решение остаётся за ним: лимит мягкий, а не жёсткий запрет.

Успех функции — пользователь замечает приближение к лимиту до того, как он превышен, и либо сдерживается, либо осознанно принимает решение потратить больше.

## 2. Functional Requirements (The "What")

### 2.1 Установка и удаление лимита

На странице детализации категории расходов, в карточке-заголовке категории, появляется отдельное поле **«Месячный лимит»**.

- Пользователь вводит сумму в том же валютном измерении, что и траты по категории.
- Лимит сохраняется **автоматически** в момент, когда пользователь покидает поле ввода.
- Если пользователь **очищает** поле — лимит снимается, кольцевой индикатор исчезает.
- Поле принимает только положительные числа больше нуля.

**Критерии приёмки:**
- [x] На странице категории «Еда» пользователь видит поле «Месячный лимит».
- [x] Пользователь вводит $500 и переходит к другому элементу — лимит сохраняется без нажатия дополнительных кнопок; вокруг иконки «Еда» на главном экране появляется кольцевой индикатор.
- [x] Пользователь очищает поле лимита и переходит к другому элементу — лимит снимается, кольцо исчезает.
- [x] При вводе нуля, отрицательного числа или текста поле показывает ошибку и лимит не сохраняется.

---

### 2.2 Кольцевой индикатор прогресса

На главном экране вокруг иконки категории отображается кольцевой индикатор прогресса — **только если для этой категории задан лимит**.

Кольцо заполняется пропорционально тому, какой процент лимита уже израсходован в текущем месяце. Цвет кольца:

| Потрачено от лимита | Цвет кольца |
|---|---|
| 0 – 60 % | Зелёный |
| 60 – 85 % | Оранжевый |
| 85 % и выше | Красный |

- Если траты превышают 100% лимита — кольцо остаётся полностью заполненным и красным.
- Категория без лимита — кольцо отсутствует, иконка выглядит как обычно.

**Критерии приёмки:**
- [x] Категория «Еда», лимит $500, потрачено $100 (20%) — кольцо зелёное, заполнено примерно на пятую часть.
- [x] Та же категория, потрачено $350 (70%) — кольцо оранжевое.
- [x] Та же категория, потрачено $470 (94%) — кольцо красное.
- [x] Та же категория, потрачено $600 (120%, превышение) — кольцо красное и полностью заполнено.
- [x] Категория без лимита — кольцо отсутствует.

---

### 2.3 Предупреждение при добавлении расхода

Когда пользователь пытается записать расход, который в сумме с уже потраченным в этом месяце превысит установленный лимит категории, — **перед подтверждением** появляется предупреждающий алерт.

- Алерт сообщает, что этот расход выведет пользователя за пределы месячного бюджета данной категории.
- Пользователь может **всё равно подтвердить** расход — тогда он записывается, кольцо становится красным.
- Пользователь может **отменить** — тогда расход не добавляется.
- Если лимита нет — никакого предупреждения не возникает.

**Критерии приёмки:**
- [x] Категория «Еда», лимит $500, потрачено $480. Пользователь пытается добавить расход $40 — появляется предупреждающий алерт о превышении месячного бюджета.
- [x] Пользователь нажимает «Подтвердить» в алерте — расход $40 записывается, итого $520, кольцо красное и полное.
- [x] Пользователь нажимает «Отмена» в алерте — расход не добавляется, итого остаётся $480, состояние кольца не меняется.
- [x] Категория «Еда» без лимита — предупреждение не появляется ни при каком размере расхода.

---

### 2.4 Ежемесячный сброс прогресса

1-го числа каждого месяца прогресс по всем категориям с лимитами сбрасывается до нуля автоматически. Кольца переходят в зелёный цвет. Сами лимиты (заданные суммы) сохраняются и продолжают действовать в новом месяце.

**Критерии приёмки:**
- [x] 31 января категория «Еда» с лимитом $500 показывала $490 потраченными — кольцо красное. 1 февраля та же категория показывает $0 потраченных — кольцо зелёное.
- [x] После сброса поле «Месячный лимит» на странице категории по-прежнему показывает $500.

## 3. Scope and Boundaries

### In-Scope

- Установка числового месячного лимита для любой категории расходов.
- Автоматическое сохранение лимита при выходе из поля ввода.
- Удаление лимита путём очистки поля.
- Кольцевой визуальный индикатор прогресса на иконке категории (главный экран), меняющий цвет по трём порогам: зелёный → оранжевый → красный.
- Предупреждающий алерт при попытке добавить расход, превышающий лимит, с возможностью всё равно подтвердить.
- Автоматический сброс прогресса в начале каждого календарного месяца; лимит при этом сохраняется.

### Out-of-Scope

- **Dashboard & Overview** — отдельный пункт роадмапа.
- **Basic Reports & Charts** — отдельный пункт роадмапа.
- Уведомления по email или push-каналу о приближении к лимиту.
- Лимиты для источников дохода или для счетов/кошельков.
- Лимиты на период, отличный от одного календарного месяца (неделя, квартал и т.д.).
- История исполнения бюджета за прошлые месяцы.
