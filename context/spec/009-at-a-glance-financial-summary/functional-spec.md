# Functional Specification: At-a-Glance Financial Summary

- **Roadmap Item:** Dashboard & Overview — At-a-Glance Financial Summary
- **Status:** Completed
- **Author:** yantsyferov

---

## 1. Overview and Rationale (The "Why")

Пользователь сегодня вынужден переходить на страницы отдельных счетов и категорий, чтобы собрать цельную картину своих финансов. Это занимает время и требует нескольких шагов. Дашборд решает эту проблему: он собирает в одном месте все ключевые финансовые показатели за выбранный месяц — сколько пришло, сколько ушло, каков итог, и куда именно уходят деньги по категориям.

**Цель:** Дать пользователю мгновенное понимание финансовой ситуации за любой месяц, не открывая другие экраны.

**Успех:** Пользователь открывает дашборд и за несколько секунд видит: заработал, потратил, разница, и на что ушли деньги.

## 2. Functional Requirements (The "What")

### 2.1. Доступ к дашборду

- В приложении появляется отдельная страница «Dashboard» (Overview).
- На неё можно перейти через навигацию (откуда именно — в рамках технической реализации, но ссылка/иконка всегда видна пользователю).

### 2.2. Выбор периода

- При открытии страницы по умолчанию отображается **текущий календарный месяц** (например, «May 2026»).
- Пользователь может листать месяцы с помощью **кнопок «‹» (назад) и «›» (вперёд)** рядом с названием месяца.
- Пользователь также может нажать на название месяца — откроется **выпадающий календарный пикер** для быстрого перехода к любому месяцу и году.
- Присутствует кнопка «**Сегодня**» (или «Текущий месяц»), которая одним нажатием возвращает отображение к текущему месяцу, если пользователь ушёл в прошлое или будущее.

**Критерии приёмки:**
- [x] При открытии страницы показывается текущий месяц.
- [x] Нажатие «‹» переключает на предыдущий месяц, «›» — на следующий. Все цифры пересчитываются.
- [x] Клик по названию месяца открывает пикер; выбор месяца в пикере немедленно обновляет все данные на странице.
- [x] Кнопка «Сегодня» возвращает к текущему месяцу и становится неактивной / скрытой, когда уже отображается текущий месяц.

### 2.3. Сводные показатели

В верхней части страницы отображаются три числа за выбранный месяц:

| Показатель | Описание |
|---|---|
| **Доходы** | Сумма всех зарегистрированных поступлений за месяц |
| **Расходы** | Сумма всех зарегистрированных трат за месяц |
| **Чистый баланс** | Доходы минус Расходы за месяц |

Дополнительно отображается одна цифра, не зависящая от выбранного периода:

| Показатель | Описание |
|---|---|
| **Общий баланс счетов** | Суммарный остаток по всем счетам и кошелькам пользователя на текущий момент |

**Критерии приёмки:**
- [x] Все четыре показателя видны на странице одновременно, чётко подписаны.
- [x] При смене месяца «Доходы», «Расходы» и «Чистый баланс» пересчитываются; «Общий баланс счетов» остаётся неизменным (он не зависит от периода).
- [x] «Чистый баланс» отображается положительным числом, если доходы превышают расходы, и отрицательным — в противном случае.

### 2.4. Разбивка расходов по категориям

Под сводными показателями располагается список категорий расходов за выбранный месяц, отсортированный по убыванию суммы трат.

Каждая строка содержит:
- Значок и название категории
- Сумму расходов по этой категории за период
- Прогресс-бар:
  - Если для категории **не установлен месячный лимит** — полоска показывает долю этой категории от общей суммы расходов за месяц (например, 40%)
  - Если для категории **установлен месячный лимит** — полоска показывает, на сколько процентов лимит израсходован, с цветовой индикацией (зелёный → оранжевый → красный по мере приближения к лимиту)

Нажатие на строку категории **переходит на детальную страницу** этой категории.

**Критерии приёмки:**
- [x] Список содержит только категории, по которым есть хотя бы одна трата за выбранный месяц. Категории с нулевыми тратами не отображаются.
- [x] Строки отсортированы от наибольшей суммы к наименьшей.
- [x] Для категории без лимита прогресс-бар заполнен пропорционально её доле в общих расходах.
- [x] Для категории с лимитом прогресс-бар показывает % использования лимита (зелёный до ~60%, оранжевый до ~85%, красный выше).
- [x] Нажатие на строку открывает страницу этой категории.

### 2.5. Пустое состояние

Если за выбранный месяц нет ни одной транзакции:
- Все суммовые показатели (Доходы, Расходы, Чистый баланс) отображаются как **$0**.
- Список категорий пуст, вместо него — поясняющий текст, например: «No transactions for this period».
- «Общий баланс счетов» показывает актуальный суммарный остаток как обычно.

**Критерии приёмки:**
- [x] При выборе месяца, в котором нет транзакций, доходы, расходы и баланс показывают $0.
- [x] Список категорий заменяется текстовым сообщением об отсутствии данных.

## 3. Scope and Boundaries

### In-Scope

- Отдельная страница дашборда с показателями за выбранный месяц
- Переключатель месяца (стрелки + пикер + кнопка «Сегодня»)
- Четыре суммовых показателя: доходы, расходы, чистый баланс, общий баланс счетов
- Список категорий расходов с прогресс-баром и переходом на детальную страницу
- Пустое состояние для периода без транзакций

### Out-of-Scope

- **Basic Reports & Charts** (Phase 4) — графики трендов по неделям и месяцам выходят за рамки этой спецификации
- **Category Breakdown Reports** (Phase 4) — расширенные отчёты по категориям за произвольный период
- **Multi-Currency Workspace** (Phase 5) — мультивалютный учёт
- **Base Value Anchor** (Phase 5) — привязка к базовой валюте / активу
- Создание или редактирование транзакций непосредственно с дашборда
- Разбивка общего баланса по отдельным счетам (только суммарная цифра)
- Разбивка доходов по источникам
